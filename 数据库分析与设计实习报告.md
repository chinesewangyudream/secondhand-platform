## 1. 引言

### 1.1 应用内容

随着互联网技术的飞速发展和人们消费观念的转变，二手交易市场逐渐成为人们处理闲置物品、实现资源循环利用的重要渠道。传统的二手交易方式存在信息不对称、交易效率低下、信任机制缺失等问题，难以满足现代人对便捷、安全、高效交易体验的需求。

二手交易平台（闲置宝）是一个基于Web的综合性二手商品交易系统，旨在为用户提供一个安全、便捷、高效的二手物品交易环境。系统支持普通商品交易和拍卖两种交易模式，集成了用户管理、商品发布与管理、在线聊天、订单管理、拍卖竞价、举报审核等核心功能，并通过引入AI智能导购功能，为用户提供智能化的商品推荐服务。

本系统采用前后端分离架构，前端使用Vue 3框架构建响应式用户界面，后端基于Spring Boot框架开发RESTful API服务，使用MySQL数据库存储系统数据，并通过WebSocket实现实时通讯功能。系统设计注重用户体验和数据安全，采用JWT令牌进行身份认证，使用Spring Security进行权限控制，确保用户信息和交易数据的安全性。

### 1.2 编写目的

本二手交易平台旨在解决以下问题：

1. **资源浪费问题**：帮助用户处理闲置物品，促进资源循环利用，减少浪费。

2. **交易信任问题**：通过用户信用评分、举报审核机制等手段，建立平台信任体系，降低交易风险。

3. **交易效率问题**：提供便捷的商品发布、搜索、沟通和交易流程，提高交易效率。

4. **价格公平问题**：引入拍卖功能，让市场决定商品价格，实现公平交易。

5. **用户体验问题**：采用现代化前端技术和友好的用户界面设计，提升用户使用体验。

本报告详细描述了二手交易平台的系统设计、数据库设计、功能实现等内容，为系统的开发、维护和扩展提供技术文档支持。

## 2. 需求分析

### 2.1 系统功能分析

#### 2.1.1 背景

在当今社会，随着人们生活水平的提高和消费观念的转变，家中积累了大量闲置物品。传统的二手交易方式主要依赖线下跳蚤市场或熟人之间的转让，存在信息传播范围有限、交易效率低下、缺乏信任保障等问题。同时，随着电商行业的发展，用户对于交易体验的要求越来越高，期望能够获得便捷、安全、透明的交易服务。

二手交易平台的出现，为用户提供了一个集中的交易场所，打破了地域限制，扩大了交易范围。通过在线发布、搜索、沟通、支付等功能，大大简化了交易流程，提高了交易效率。同时，平台通过建立信用体系、举报审核机制等手段，有效保障了交易安全。

#### 2.1.2 功能描述

**a. 用户管理功能**
- 用户注册：新用户可通过用户名、邮箱、手机号等信息注册账户
- 用户登录：已注册用户可通过用户名和密码登录系统
- 个人信息管理：用户可修改个人资料、头像、密码等信息
- 收货地址管理：用户可添加、修改、删除收货地址

**b. 商品管理功能**
- 商品发布：用户可发布闲置商品，包括标题、描述、价格、分类、图片等信息
- 商品编辑：用户可修改已发布商品的信息
- 商品下架：用户可将商品下架或重新上架
- 商品搜索：用户可通过关键词、分类、价格区间等条件搜索商品
- 商品详情：展示商品的详细信息、卖家信息、相关推荐等

**c. 收藏功能**
- 商品收藏：用户可收藏感兴趣的商品
- 收藏管理：用户可查看和管理收藏列表

**d. 在线聊天功能**
- 即时通讯：买卖双方可通过WebSocket实时聊天
- 消息记录：系统保存聊天记录，方便用户查看历史消息
- 会话管理：用户可管理聊天会话列表

**e. 订单管理功能**
- 创建订单：买家可对商品创建订单
- 订单支付：买家可对订单进行支付
- 订单发货：卖家可填写物流信息发货
- 订单确认：买家确认收货完成交易
- 订单取消：在特定状态下可取消订单

**f. 拍卖功能**
- 创建拍卖：卖家可将商品设置为拍卖模式
- 出价竞拍：买家可对拍卖商品出价
- 拍卖结束：拍卖时间结束后，最高出价者获得商品

**g. 举报审核功能**
- 商品举报：用户可举报违规商品
- 举报审核：管理员审核举报并处理违规商品

**h. AI智能导购**
- 智能推荐：基于AI技术为用户推荐合适的商品

#### 2.1.3 用户功能

**a. 用户注册与登录**
- 提供用户名、邮箱、密码等信息完成注册
- 支持用户名或邮箱登录
- 记住登录状态功能

**b. 商品浏览与搜索**
- 首页展示最新上架商品和热门分类
- 支持按分类、关键词、价格区间搜索
- 商品详情页展示完整信息和卖家信息

**c. 商品发布与管理**
- 填写商品标题、描述、价格、分类、成色等信息
- 上传商品图片
- 管理已发布的商品（编辑、下架）

**d. 收藏商品**
- 点击收藏按钮收藏商品
- 在个人中心查看收藏列表

**e. 在线聊天**
- 与卖家/买家实时沟通
- 发送文字、图片消息
- 查看聊天历史记录

**f. 订单交易**
- 创建订单并支付
- 填写收货地址
- 卖家发货填写物流信息
- 买家确认收货

**g. 参与拍卖**
- 浏览拍卖商品
- 出价竞拍
- 查看竞拍记录

**h. 举报违规商品**
- 选择举报原因
- 填写举报描述

**i. 管理员审核**
- 审核商品发布
- 处理用户举报
- 管理违规商品

#### 4.2.3 设计与实现技术

本系统采用前后端分离架构，后端基于Spring Boot框架开发RESTful API服务，前端基于Vue 3框架构建单页面应用。

**后端架构：**

系统采用分层架构设计，分为Controller层、Service层、Mapper层和Entity层：

- **Controller层**：处理HTTP请求，进行参数校验和响应封装
- **Service层**：实现业务逻辑，处理事务
- **Mapper层**：使用MyBatis Plus进行数据库操作
- **Entity层**：定义实体类，映射数据库表

**安全设计：**

- 使用Spring Security进行权限控制
- 使用JWT进行身份认证
- 密码使用BCrypt加密存储

**实时通信：**

- 使用Spring WebSocket实现实时聊天
- 支持SockJS降级方案
- 使用STOMP协议进行消息传递

**前端架构：**

- 使用Vue 3 Composition API编写组件
- 使用Pinia进行全局状态管理
- 使用Vue Router进行路由管理
- 使用Axios进行HTTP请求
- 使用Element Plus构建UI界面

#### 4.4.1 数据库配置文件

```yaml
# application.yml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/secondhand_db?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: your_password

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```
